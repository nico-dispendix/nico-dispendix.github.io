Subscribe_DeviceInputPressure_Parameters

Parameters for DeviceInputPressure

Subscribe_DeviceInputPressure_Responses

FieldTypeLabelDescription
DeviceInputPressure sila2.org.silastandard.Real

Input pressure of the device. A minimum of 2.6 bar is needed for dispensing. If the input pressure is below 3 bar and a dispense is started, the device will stay in idle until sufficient input pressure is applied. If the input pressure drops below 2.6 bar during dispense, the dispense will stop and be continued when and if sufficient input pressure is applied.

InstrumentStatusProvider

Provider to access properties related to the I.DOT status. This provider can be used to check the state of the device before executing another command in order ensure proper functioning. It can also be used for maintenance and troubleshooting processes.

PlateMethodService

Service to eject and retract both source and target plates. This service is needed to input the respective target and source plates before dispensing. Please note that canceling this service in the middle of execution can cause the carrier to half eject or half retract. The dispensing process needs all carriers to be fully retracted to avoid any harm to the I.DOT.
